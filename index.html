About

What this simulator shows

Compact reference text for rate limiting, queue pressure, and failure behavior.

This rate limiter simulator models how incoming traffic moves through a fast rate-limiter decision path, app pending capacity, and a downstream dependency. It is designed to make HTTP 429 rate-limit rejects, HTTP 503 capacity failures, queue buildup, pending requests, latency distribution, and bottleneck behavior easier to understand in a distributed system.

The simulator supports fixed-window and sliding-window thinking for policy behavior, and it highlights how limiter rules protect backend capacity by reducing overload before app timeouts and downstream saturation create visible failures.
